# 应用页面使用指南

## 📋 概述

"应用"页面用于展示您开发的所有项目和应用程序，替代了原来的"联系"页面。

## 🎯 页面特性

### ✨ 主要功能
- **分类展示** - 专业项目、开源项目、实用工具、研究项目四大类别
- **智能筛选** - 按技术领域快速过滤项目
- **统计信息** - 实时显示各类项目数量
- **响应式设计** - 完美适配桌面、平板、手机
- **精选标记** - 突出展示重点项目

### 🎨 视觉设计
- **玻璃拟态卡片** - 现代化的毛玻璃效果
- **渐变色彩** - 统一的品牌色系
- **悬停动效** - 丰富的交互反馈
- **状态徽章** - 不同颜色标识项目状态
- **技术标签** - 清晰展示技术栈

## 📝 添加新应用/项目

### 编辑配置文件

打开 `_data/applications.yml`，根据项目类型在对应部分添加：

```yaml
# 专业项目示例
professional:
  - name: "项目名称"
    description: "项目简介，一两句话描述项目核心功能"
    category: "项目分类（如：医疗机器人、医学图像处理等）"
    tech_stack: ["技术1", "技术2", "技术3"]  # 技术栈列表
    status: "项目状态"  # 可选：生产环境、研究中、开发中、稳定、活跃维护
    highlight: true     # 是否为精选项目（true/false）
    features:           # 主要特性列表
      - "特性1"
      - "特性2"
      - "特性3"
    link: "https://项目链接"      # 项目主页（可选）
    github: "https://github链接"  # GitHub仓库（可选）
    demo: "https://演示链接"      # 在线演示（可选）
```

### 项目分类说明

**1. professional（专业项目）**
- 用于展示商业项目、工作项目
- 通常是生产环境或高完成度项目
- 建议设置 `highlight: true` 突出显示

**2. opensource（开源项目）**
- 个人开源项目
- 开源贡献
- 建议提供 GitHub 链接

**3. tools（实用工具）**
- 开发工具
- 辅助脚本
- 实用小程序

**4. research（研究项目）**
- 学术研究项目
- 探索性项目
- 实验性功能

### 状态标识说明

| 状态 | 说明 | 徽章颜色 |
|------|------|---------|
| 生产环境 | 已上线使用的项目 | 绿色渐变 |
| 研究中 | 正在研究开发 | 蓝色渐变 |
| 开发中 | 正在开发中 | 橙色渐变 |
| 稳定 | 功能稳定 | 紫色渐变 |
| 活跃维护 | 持续维护更新 | 青色渐变 |

### 筛选类别

当前支持的筛选类别（可在 `apps.html` 中自定义）：
- 医疗机器人
- 医学图像处理
- 3D视觉
- Web开发
- 工具库
- 学术研究

**添加新类别：**
1. 在 `applications.yml` 中为项目设置新的 `category`
2. 在 `apps.html` 的筛选器部分添加对应按钮

## 🎨 样式定制

### 修改配色

编辑 `assets/styles.scss`，找到应用页面样式部分：

```scss
/* 修改主题色 */
.apps-main-title {
    background: linear-gradient(135deg, #3B82F6, #8B5CF6, #EC4899);
}

/* 修改卡片样式 */
.app-card {
    background: var(--glass-bg);
    border-radius: 16px;
}

/* 修改技术标签颜色 */
.tech-stack-tag {
    background: rgba(59, 130, 246, 0.1);
    color: var(--accent-color);
}
```

### 自定义卡片布局

在 `_includes/app-card.html` 中修改卡片组件结构。

## 📊 页面元素说明

### 统计卡片
自动统计各类别项目数量，无需手动更新。

### 筛选器
- **全部** - 显示所有项目
- **分类按钮** - 按技术领域筛选
- 点击按钮即时筛选，带淡入动画

### 应用卡片
每个卡片包含：
- **标题栏** - 项目名称 + 精选标记 + 状态徽章
- **分类** - 项目所属领域
- **描述** - 项目简介
- **特性列表** - 主要功能特性
- **技术栈** - 使用的技术和工具
- **链接按钮** - 项目主页、GitHub、在线演示

## 💡 最佳实践

### 项目描述
- 保持简洁，1-2句话
- 突出核心价值
- 使用动词开头

### 特性列表
- 每项特性简短明确
- 3-4个核心特性即可
- 突出技术亮点

### 技术栈
- 列出主要技术
- 控制在3-6个
- 按重要性排序

### 链接
- 确保链接有效
- 优先提供可访问的演示
- GitHub链接增加可信度

## 🔄 更新流程

1. **编辑配置**
   ```bash
   # 编辑应用数据
   编辑 _data/applications.yml
   ```

2. **保存并查看**
   - Jekyll 会自动重新生成页面
   - 刷新浏览器即可看到更新

3. **提交更改**
   ```bash
   git add _data/applications.yml
   git commit -m "更新应用列表"
   git push
   ```

## ⚠️ 注意事项

1. **YAML 格式** - 注意缩进和引号
2. **链接有效性** - 定期检查链接是否失效
3. **图标显示** - 链接图标使用内联SVG
4. **性能优化** - 项目过多时考虑分页
5. **SEO优化** - 确保 description 字段准确

## 🎯 常见问题

### Q: 如何更改筛选器类别？
A: 编辑 `apps.html` 第 41-48 行的筛选按钮列表。

### Q: 如何隐藏某个区块？
A: 在 `apps.html` 中注释掉对应的 section。

### Q: 卡片链接不显示？
A: 检查 `applications.yml` 中的 link/github/demo 字段是否为空字符串或空值。

### Q: 如何调整卡片大小？
A: 修改 `assets/styles.scss` 中的 `.apps-grid` 的 `minmax(350px, 1fr)` 值。

### Q: 如何添加新的项目状态？
A: 在 `assets/styles.scss` 中添加对应的 `.app-badge-状态名` 样式类。
